How long does validation take?

A metadata file will usually take up to 20 seconds to a minute to finish validating, which is usually taken up by the time it takes to complete XSD validation.

  • Why do I have to validate my metadata files?

    Metadata files are validated to ensure that the e-Science Centre will be able to process and display the contents of each file and that links to ontology terms and other metadata registrations will work as expected.

  • What conditions are required to pass validation?

    Metadata files must meet these conditions to pass validation:

    • The file uses valid XML syntax.
    • The file uses UTF-8 encoding (and does not contain illegal characters).
    • The file name is the same as the local ID.
    • The file uses the correct type for the file upload category (e.g. the metadata file should have <Organisation> as the root element, if an organisation is being registered/updated).
    • The file is structured according to the rules and constraints defined in the relevant XSD schemas (see the Space Physics Metadata Model).
    • The file has a unique and appropriately formatted local ID (see Chapter 4.1.1 Local ID of the Data Resource Registration Guide for further guidance).
    • The file has an appropriately formatted namespace (Chapter 4.1.2 Choice of namespace of the Data Resource Registration Guide gives further guidance).

    Any URL references to other metadata registrations and/or Space Physics Ontology terms in the metadata file are also checked to ensure they point to registrations/terms currently registered in the e-Science Centre/Space Physics Ontology.

    Additional checks are performed depending on if you are registering or updating.
